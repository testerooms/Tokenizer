Privacy Policy

Last updated: May 25, 2026

1. What We Collect

Tokenizer is a self-hosted open-source proxy. When you run it on your infrastructure, it processes API requests routed through the proxy. The proxy logs the following metadata for budget tracking purposes:

  • Engineer ID (provided via the x-engineer-id header)
  • Number of input/output tokens per request
  • Model name used for each request
  • Timestamp of each request
  • HTTP status code of the proxied response

The proxy does not log, store, or transmit the content of any prompts, responses, or API request bodies. Token payloads are anonymized — we count tokens, we do not read them.

2. Landing Page (tokenizer.sh)

The marketing site hosted at tokenizer.sh uses Vercel Analytics for anonymous traffic measurement. This collects:

  • Page views and referrer information
  • Browser type and device category
  • Geographic region (country-level only)

No cookies are used for analytics. No personal identifiable information is collected. You can opt out by disabling JavaScript or using a browser extension.

3. Data Storage

All usage data is stored locally in a SQLite database on the machine running the proxy. No data is sent to external servers unless you explicitly configure an alert integration (e.g., Slack webhook). You retain full control over your data.

4. Data Retention

Usage logs are retained indefinitely by default. You can clear the database at any time by deleting the guardian.db file or running the appropriate SQL command. We recommend setting up a retention policy appropriate for your organization.

5. Third-Party Services

The proxy forwards API requests to Anthropic's API. Data sent to Anthropic is subject to Anthropic's Privacy Policy. No other third-party services receive data from the proxy.

6. GDPR Compliance

If you are processing data of individuals in the European Economic Area, you are the data controller and Tokenizer is a data processing tool. You are responsible for:

  • Ensuring you have a lawful basis for processing
  • Implementing appropriate technical measures (the proxy is one such measure)
  • Responding to data subject access requests

7. Security

The proxy does not expose data to the public internet by default. It is designed to run on internal networks. We recommend deploying behind a reverse proxy with authentication, TLS, and network access controls appropriate for your organization's security requirements.

8. Changes

We may update this policy as the project evolves. Changes will be posted here. Continued use after changes constitutes acceptance of the updated policy.

9. Contact

For questions about this policy, open an issue on GitHub.
